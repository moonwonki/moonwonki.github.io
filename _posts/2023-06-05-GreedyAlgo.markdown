# 그리디 알고리즘이란?

그리디 알고리즘은 최적해를 구하는 데 사용되는 알고리즘 디자인.

욕심장이 기법이라고도 불린다.

이 알고리즘은 매순간마다 가장 최선의 선택을 하는 탐욕적인 방법을 사용하여 문제를 해결한다.

<br/>

**모든 연산이 끝나지 않았더라도 그 순간 고를 수 있는 최적의 해를 구하는 것이 목표다.**

<br/>

어떻게 보면 굉장히 당연한 말이다. 

그러나 자세히 살펴보면 설계가 굉장히 중요한 기법인데,

그저 눈앞에 있는 최적의 값만을 모으는 것이 전체 문제를 해결하는데 악수가 될 수도 있기 때문이다.

<br/>

따라서 **매 선택의 순간마다 선택한 최적의 해가 점차적으로 축적되어 전체 문제 해결의 도움이 되도록 알고리즘 구조를 설계해야 한다.**

> 그리디 알고리즘을 사용할 때에는 문제의 특성과 제약 조건을 고려하여 정확한 해결 방법인지 확인하는 것이 중요하다!

<br/>

<br/>

## 대표적인 예시 - 분할 배낭 문제

한 개의 가방 안에 최대한 좋은 물건들을 담는 문제이다.

<br/>

**무게 제한이 40인 가방 안에 최대한 비싼 물건들을 담고자 한다. **

**각 물건들의 무게와 가격, 개수가 주어질 때, 배낭에 담은 물건들의 가격의 합이 최대가 되려면 어떻게 해야할까?** 

**(물건을 잘라서 담는 것이 가능하다!)**

<br/>

| 물건 이름 | 무게 | 가격 | 개수 |
| :------ |:--- | :--- | :--- |
| 치즈 | 5 | 10 | 3 |
| 케이크 | 15 | 20 | 1 |
| 파이 | 20 | 10 | 2 |

<br/>

### 해결 방안

그리디 알고리즘 기법으로 접근해 보자.

지금 내릴 수 있는 최고의 선택은 바로 "무게 대비 비싼 물건을 담는 것"이다.

가격 / 무게로 비교해 보았을 때, 치즈는 2, 케이크는 4/3, 파이는 1/2이다.

따라서 지금 최적의 물건은 치즈다.

치즈는 3개밖에 없으므로 일단 치즈를 3개 담아보자.

**(현재 상황 : 남은 무게 - 25 / 현재 가격 : 30)**

케이크 다음으로 최적의 물건은 케이크이다.

케이크는 1개밖에 없다. 케이크를 담아보자

**(현재 상황 : 남은 무게 - 10 / 현재 가격 : 50)**

그 다음 남은 거는 파이다. 남은 무게가 10인데 파이는 20...? 괜찮다. 반으로 나눠서 가져가면 된다.

가치도 반으로 떨어지겠지만, 빈 공간을 채워서 최대한 가져가는게 좋지 않은가?

문제 조건에도 '물건을 잘라서 담는 것이 가능하다'고 나와있다.

**(현재 상황: 남은 무게 - 0 / 현재 가격 : 55)**



<br/>

<br/>

<br/>

위 방식을 보면 매 선택의 순간마다 가장 가성비 좋은 즉, 최적의 선택을 하는 것을 알 수 있다.

내가 이 개념을 처음 봤을 때는 "당연한거 아니야? 누가 바보같이 가치가 떨어지는 걸 먼저 담아"라고 생각했는데,

DP를 배우면서 이 방식이 반드시 먹히지 않는다는 것을 깨달았다.

저기서 그리디 알고리즘이 가능한 이유는 '물건을 잘라서 담는 것이 가능하'기 때문이다.

만약 물건을 잘라서 담는 것이 불가능하다면, 우리는 단순히 가격 / 무게를 절대적인 기준으로 삼을 수 없다.

<br/>

**가방 무게 : 100**  (물건을 잘라서 담는 것은 불가능하다!)

| 물건 이름 | 무게 | 가격 | 개수 |
| --------- | ---- | ---- | ---- |
| A         | 51   | 70   | 5    |
| B         | 50   | 50   | 5    |

<br/>

위 문제의 경우 가격 / 무게가 가장 높은 A를 먼저 담았다가는 망한다.

이렇듯 문제에서 주어진 조건과 설계가 그리디에 적합하지 않는다면, 다른 방식으로 문제를 풀어야 한다.
